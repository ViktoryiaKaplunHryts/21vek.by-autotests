# 🛒 Автоматизация тестирования 21vek.by

Фреймворк для автоматизированного тестирования функционала интернет-магазина [21vek.by](https://21vek.by).  
Реализован с использованием паттерна **Page Object Model (POM)**.

---

### ⚠️ Статус проекта: В разработке
В текущей версии представлены **не все запланированные тесты**. Ведётся работа над расширением тестового покрытия.

---

## 🎯 Цель проекта
Получение практического опыта автоматизации тестирования с использованием **Java**, **Selenium** и **TestNG**, а также настройка процессов непрерывной интеграции (CI/CD).

---

## 🛠 Стек технологий
* **Java** - язык написания автотестов
* **Selenium WebDriver** - автоматизация браузера
* **TestNG** - тестовый фреймворк
* **Allure** - формирование отчётов
* **WebDriverManager** - управление драйверами
* **Maven** - сборка проекта и управление зависимостями
* **Jenkins** - CI/CD сервер для запуска тестов

## ⚙️ Конфигурация и безопасность

### 🔐 Тестовые данные
Все конфиденциальные данные (логины, пароли, номера телефонов) вынесены в отдельный файл **`test-data.properties`**. Файл добавлен в `.gitignore`

---

### 🔄 Непрерывная интеграция (CI/CD)
Проект полностью интегрирован с **Jenkins**:

**Безопасное окружение:** На сервере Jenkins создана локальная копия файла `test-data.properties` с реальными тестовыми данными. Во время сборки Jenkins использует именно этот локальный файл, что обеспечивает корректный запуск тестов в защищённой среде.

**Отчёты о запуске:** После каждого запуска тестов в Jenkins автоматически генерируется отчёт **Allure**, который доступен для просмотра и анализа прямо в интерфейсе Jenkins.
